Nie mam pojęcia, jak zacząć rozwiązywać tę misję.

Proszę złapać pomysł 💡! Proszę utworzyć dwie zagnieżdżone pętle iterujące po indeksach, jedną od początku podanego ciągu a, drugą - od końca. Proszę wziąć podciąg pomiędzy tymi indeksami i sprawdzić, czy jest palindromem. Jeśli tak, proszę porównać z predefiniowanym ciągiem wynikowym - jeśli długość bieżącego ciągu jest większa, proszę zaktualizować wartość ciągu wynikowego i zwrócić ją na końcu funkcji.

Ok, od czego powinienem zacząć?

Obliczmy długość podanego ciągu za pomocą funkcji len() i zdefiniujmy zmienną res, aby przechowywać najdłuższy podciąg palindromiczny z początkową wartością pustego ciągu. 

l = len(a)
res = ""
Co robić dalej?

Proszę użyć pętli for do iteracji po wartościach indeksu początkowego, który może być dowolny w podanym ciągu, a drugiej pętli do iteracji po wartościach indeksu końcowego, który może być następny po początkowym (aby uwzględnić tylko znak o indeksie początkowym) do następnego po długości podanego tekstu (aby uwzględnić tylko ostatni znak). 

for s in range(l):
    ...
    for e  in range(s + 1, l + 1):
        ...
Proszę mi pomóc.

Proszę się zastanowić. Jeśli znajdzie Pan już jakiś podciąg palindromiczny o długości 5 (na przykład), a indeks początkowy znajduje się już 3 znaki od końca podanego ciągu, to na pewno nie znajdzie Pan dłuższego palindromu. Proszę to wykorzystać, aby zwiększyć wydajność swojego rozwiązania, porównując wspomniane wartości i if, aby przerwać zewnętrzną pętlę. 

for s in range(l):
    if l - s <= len(res):
        break
Co zrobić z indeksami?

Wewnątrz wewnętrznej pętli proszę znaleźć podciąg przy użyciu ciągu slicing i bieżących indeksów. 

subs = a[s:e]

Teraz proszę sprawdzić, czy ten ciąg jest palindromiczny (używając odwróconego cięcia pełnego ciągu z ujemnym krokiem) i czy jego długość jest większa niż wynikowego ciągu. 

if subs == subs[::-1] and len(subs)> len(res):
    ...

Jeśli tak, proszę zaktualizować wartość wynikowego ciągu.

res = subs
Jak powinno wyglądać moje ostateczne rozwiązanie?

Wystarczy połączyć wszystkie poprzednie podpowiedzi, zwrócić wynikowy ciąg znaków i zawinąć kod w wyrażenie funkcyjne. Używanie podpowiedzi typu dla danych wejściowych i wyjściowych jest wysoce zalecane.

Poniżej znajduje się całe rozwiązanie:

def longest_palindromic(a: str) -> str:

    l = len(a)
    res = ""
    for s in range(l):
        if l - s <= len(res):
            break
        for e  in range(s + 1, l + 1):
            subs = a[s:e]
            if subs == subs[::-1] and len(subs)> len(res):
                res = subs

    return res

Oto inne podejście: nadal podwójna pętla, ale jedna jest dla długości podciągu (od większej, aby była wydajna), a druga - dla indeksu początkowego. Następnie proszę znaleźć podciąg i sprawdzić, czy jest palindromiczny. Jeśli tak, proszę go natychmiast zwrócić. 

def longest_palindromic(text):

    s = len(text)
    for size in range(s)[::-1]:
        for index in range(s - size):
            word = text[index:index + size + 1]
            if word == word[::-1]:
                return word

Proszę sprawdzić wszystkie możliwe ciągi za pomocą rekurencji, wywołując tę samą funkcję dla dwóch ciągów bez jednego znaku granicznego i wybrać najdłuższy palindromiczny. 

def longest_palindromic(text):

    if text == text[::-1]:
        return text

    return max(longest_palindromic(text[:-1]), longest_palindromic(text[1:]), key = len)
